🔄 完整交互流程

💧 添加水

触发: 点击 + 按钮
动画: 数字弹跳 (scale 1.2 → 1.0, 0.3s) + 水位平滑上升 (1.2s ease-out)
记录: 自动记录时间戳到历史
目标达成: 振动反馈 + 庆祝动画 (confetti) + "目标达成！" 提示
上限: 最多添加到目标数，不可超出

🗑️ 删除记录

移动端: 左滑记录项 → 显示删除按钮 → 点击确认删除
手表端: 长按记录项 → 弹出确认对话框 → 确认删除
动画: 记录项滑出 + 数字减少 + 水位下降
撤销: 删除后显示 3 秒撤销提示，点击可恢复

📊 视图切换

触发: 点击左侧视图按钮
弹出菜单: 日视图 / 周视图 / 月视图
按钮状态: 显示当前视图名称，展开时变为 × 关闭
切换动画: 淡入淡出 (0.2s)
日视图: 显示当天记录列表，支持日期导航
周视图: 显示本周柱状图，当天高亮
月视图: 显示日历网格，颜色表示完成度

深色模式: 开关切换，立即生效，全局主题变化

每日目标: 点击弹出数字选择器 (1-20杯)，确认后更新
退出: 点击底部返回按钮 → 页面滑出

📐 统一元素大小设计原则

🎯 核心原则：最小触摸区域 44×44px

• 所有可交互元素的最小触摸区域不小于 44×44px（Apple HIG 标准）
• 主要操作按钮建议 56-64px 直径，确保易于点击
• 次要按钮最小 44px 直径
• 按钮间距至少 8px，避免误触
组件 手机竖屏 手机横屏 平板 手表
主数字 64px 48px 96px 40-44px
添加按钮 ⭐ 56-64px 48px 64-96px 56-64px
视图/菜单按钮 44px 36px 48px 40px
Tab栏高度 40px 32px 48px -
Tab项最小宽度 44px 36px 48px -
底部安全距离 24px 16px 24px 8px

💡 设计要点

添加按钮：作为主要操作，使用较大尺寸（56-64px），配合自发光效果突出
视图按钮：次要操作，使用标准最小尺寸（44px），位于左侧便于拇指操作
菜单按钮：次要操作，使用标准最小尺寸（44px），位于右侧
手表适配：虽然屏幕小，但交互元素仍需保持足够大（56-64px），减少误触
横屏适配：左右分屏布局，Tab栏隐藏，主卡片始终可见
